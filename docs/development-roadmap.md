# 개발 WBS 및 로드맵

이 문서는 Stock Alarm의 남은 개발건을 작업 단위로 쪼개고, 어떤 순서로 진행할지 보기 위한 WBS입니다.

날짜 기준: 2026-06-24

## 원칙

- 최종 사용처는 개인 PC에서 로컬 서버를 실행하고, 텔레그램으로 상태 확인과 알림을 받는 흐름입니다.
- Postgres 전환, 앱스토어/플레이스토어 출시, Railway 상시 배포는 더 이상 진행하지 않습니다.
- 데이터 저장은 로컬 JSON 파일과 자동 백업을 기본으로 유지합니다.
- 모바일/배포/스토어 관련 코드는 즉시 삭제하지 않고, 테스트 영향과 백업 기준을 확인한 뒤 단계적으로 정리합니다.
- 무료/공개 API는 실패 가능성이 있으므로, 사용자가 실패 이유를 볼 수 있는 진단 기능을 먼저 강화합니다.
- 예상 기간은 1명이 순차 개발한다고 가정한 상대적인 작업량입니다. 실제 기간은 테스트 범위와 API 접근 가능 여부에 따라 달라집니다.

## 방향 전환 결정

2026-06-22 기준으로 프로젝트 방향을 개인용 로컬 운영으로 고정합니다. 앞으로는 PC 실행, 텔레그램 명령, 웹 대시보드, JSON 백업/복구, 시세/배당 진단을 핵심 기능으로 보고, 앱 출시나 서버 배포를 위한 코드는 정리 후보로 관리합니다.

유지할 축:

- `start-local.bat`, `status-local.bat`, `stop-local.bat` 중심의 Windows PC 운영
- 웹 대시보드와 관리자 화면
- 텔레그램 알림, 텔레그램 명령, 일일 브리핑
- 로컬 JSON 저장소, 자동 백업, CSV 가져오기/내보내기
- KIS/Naver/공공데이터/배당 provider 진단과 실패 사유 표시

정리 후보:

| 구분 | 대상 | 판단 | 정리 방식 |
|---|---|---|---|
| Postgres 전환 | `src/postgresStore.js`, `src/postgresMigrationDryRun.js`, Postgres 테스트 fixture, `migrate:postgres:*` 스크립트 | 개인 로컬 운영에는 불필요 | WBS 13.3에서 소스 제거, 설계 문서는 `docs/archive/` 보관 |
| 앱 출시 | `mobile/`, `/api/mobile/*`, Expo Push, 앱 심사/스토어 자산 문서와 점검 CLI | 앱 출시 계획 없음 | 모바일 기능을 보관 후보로 표시하고 서버 API 의존성 제거 |
| 배포 | `railway.json`, `.env.railway.example`, Railway 문서와 점검 CLI | 상시 배포 계획 없음 | README에서 운영 경로에서 제외하고 별도 보관 또는 삭제 |
| 원격 푸시 | Expo Push 토큰, 모바일 푸시 전송 경로 | 텔레그램으로 충분 | 가격/배당 알림 경로를 텔레그램 중심으로 단순화 |
