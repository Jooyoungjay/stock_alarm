# 개발 WBS 및 로드맵

이 문서는 Stock Alarm의 남은 개발건을 작업 단위로 쪼개고, 어떤 순서로 진행할지 보기 위한 WBS입니다.

날짜 기준: 2026-05-26

## 원칙

- 지금은 로컬 PC 실행 기반 MVP를 안정화하는 것이 우선입니다.
- 앱스토어/플레이스토어 출시는 당장 배포보다 데이터 구조와 모바일 API 기반을 먼저 다진 뒤 진행합니다.
- 무료/공개 API는 실패 가능성이 있으므로, 사용자가 실패 이유를 볼 수 있는 진단 기능을 먼저 강화합니다.
- 예상 기간은 1명이 순차 개발한다고 가정한 상대적인 작업량입니다. 실제 기간은 테스트 범위와 API 접근 가능 여부에 따라 달라집니다.

## 상태 기준

| 상태 | 의미 |
|---|---|
| 예정 | 아직 착수하지 않은 작업 |
| 진행중 | 일부 구현 또는 진행 중인 작업 |
| 완료 | 개발, 문서, 테스트 반영 완료 |
| 보류 | 외부 API, 정책, 후속 결정 대기 |
