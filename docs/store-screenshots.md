# 스토어 스크린샷 제작 가이드

날짜 기준: 2026-05-20

이 문서는 Stock Alarm을 App Store와 Google Play에 제출할 때 사용할 스토어 스크린샷 화면, 문구, 데모 데이터 기준을 정리합니다. 실제 제출 직전에는 아래 공식 문서의 최신 해상도와 파일 형식을 다시 확인합니다.

## 공식 기준

- Apple Screenshot specifications: https://developer.apple.com/help/app-store-connect/reference/app-information/screenshot-specifications/
- Google Play preview assets: https://support.google.com/googleplay/android-developer/answer/9866151

현재 앱은 `mobile/app.json`에서 `ios.supportsTablet=true`로 설정되어 있으므로 App Store 제출 시 iPhone뿐 아니라 iPad 스크린샷도 준비해야 합니다. iPad를 지원하지 않기로 결정하면 앱 설정과 심사 전략을 먼저 바꾼 뒤 스크린샷 범위를 다시 정합니다.

## 제출 세트

| 스토어 | 우선 제작 세트 | 기준 |
|---|---|---|
| App Store | iPhone portrait 6장 | Apple은 기기군별 1~10장 제출을 허용합니다. iPhone은 6.9형 portrait 해상도를 우선 준비하고, 필요 시 6.5형 이하를 보강합니다. |
| App Store | iPad portrait 4장 | 현재 iPad 지원 설정이 켜져 있으므로 13형 iPad portrait 세트를 준비합니다. |
| Google Play | Android phone portrait 6장 | 앱 추천 노출까지 고려해 9:16, 1080x1920 이상, 4장 이상을 기본으로 합니다. |

## 데모 데이터 원칙

- 실제 사용자 개인정보, 실제 계좌 금액, 실제 투자 메모를 사용하지 않습니다.
- 샘플 종목은 국내/미국 종목을 섞되, 모든 금액은 데모 데이터임을 내부 캡처 파일명과 작업 메모에 남깁니다.
- 텔레그램 봇 토큰, API 키, 서버 URL, deviceSecret, 개인 이메일이 화면에 보이면 캡처하지 않습니다.
- 상태바에는 통신사명, 개인 알림, 낮은 배터리, 실명 계정 정보가 노출되지 않게 합니다.
- 수익 보장처럼 보이는 문구는 넣지 않고, 앱이 알림/기록 도구라는 점만 보여줍니다.

## 캡처 화면

| 순서 | 파일명 | 화면 | 제목 문구 | 설명 문구 | 대체 텍스트 |
|---:|---|---|---|---|---|
| 1 | `01-portfolio-summary` | 포트폴리오 요약 | 내 계좌 상황 한눈에 | 등록, 활성, 위험, 알림 수를 첫 화면에서 확인합니다. | 포트폴리오 요약과 위험 종목 개수가 표시된 Stock Alarm 첫 화면 |
| 2 | `02-watchlist-risk` | 감시 종목 목록 | 팔아야 할 기준을 숫자로 | 현재가, 기준가, 최고가 대비 하락률, 이익금 반납률을 함께 봅니다. | 두산퓨얼셀과 리노공업 감시 종목 카드에 기준가와 반납률이 표시된 화면 |
| 3 | `03-stock-form` | 종목 등록/편집 | 평단가와 매도 기준을 한 번에 | 평단가, 수량, 알림 기준, 매수 이유와 매도 조건을 한 화면에서 입력합니다. | 종목 코드, 매수가, 보유 수량, 알림 기준을 입력하는 등록 화면 |
| 4 | `04-alert-toggle-push` | 알림 토글/푸시 | 알림은 켜고 끄기 쉽게 | 종목별 알림 ON/OFF와 테스트 푸시로 수신 상태를 확인합니다. | 종목 카드의 알림 켜기 버튼과 테스트 푸시 버튼이 보이는 화면 |
| 5 | `05-dividend-calendar` | 배당 캘린더 | 배당 일정까지 같이 확인 | 예상 지급월, 배당락일, 지급일, 월별 예상 현금흐름을 확인합니다. | 월별 배당 일정과 예상 배당금 합계가 표시된 배당 캘린더 화면 |
| 6 | `06-alert-history` | 알림 기록 | 받은 알림을 다시 확인 | 가격 알림과 배당 이벤트 알림의 전송 시각과 사유를 확인합니다. | 최근 가격 알림과 배당 이벤트 알림 기록이 목록으로 표시된 화면 |

현재 모바일 앱은 서버 연결, 포트폴리오 요약, 배당 캘린더, 알림 기록, 종목 등록/편집, 알림 토글, 푸시 테스트 흐름을 갖고 있습니다. 실제 스토어 제출 전에는 샘플 데이터가 들어간 HTTPS 데모 서버와 실제 기기 캡처 파일을 준비합니다.

## 화면별 데모 값

| 항목 | 값 |
|---|---|
| 국내 종목 | 두산퓨얼셀 `336260`, 리노공업 `058470` |
| 우선주 예시 | 두산퓨얼셀우 `33626L` |
| 미국 종목 | Apple `AAPL`, Tesla `TSLA` |
| 알림 기준 | 이익금 반납률 10%, 최고가 대비 하락률 8%, 직접 기준가 |
| 배당 예시 | 분기 배당, 지급월 `3,6,9,12`, 예상 연 배당금 표시 |
| 푸시 예시 | 테스트 푸시 전송 완료, 가격 조건 도달 알림 |

## 제작 체크리스트

- App Store Connect와 Play Console에 올리기 전 공식 문서 기준으로 해상도와 파일 형식을 다시 확인합니다.
- iPhone, iPad, Android phone 세트의 첫 3장은 실제 앱 UI가 중심이 되게 구성합니다.
- 추가 문구를 넣는 경우 화면의 20%를 넘지 않게 하고, 작은 글씨로 과밀하게 만들지 않습니다.
- `Best`, `1위`, `수익 보장`, `지금 설치` 같은 순위/성과/호출 문구를 쓰지 않습니다.
- 모든 스크린샷에 한국어 문구와 140자 이하의 대체 텍스트를 준비합니다.
- 캡처 후 `npm run check:demo`로 심사용 URL과 푸시/텔레그램 시연 조건을 다시 확인합니다.
