# 사용자/관리자 페이지 분리 설계

날짜 기준: 2026-05-15

현재 구현 상태:

- `/`와 `/app`은 사용자 화면으로 동작합니다.
- `/admin`은 관리자 화면으로 동작합니다.
- 서버 상태, 시세/배당 provider 진단, KIS 현재가 점검, 백업/복구/삭제, 개발 WBS는 관리자 화면으로 이동했습니다.
- 모바일 하단 탭은 사용자 화면에서만 보입니다.
- `ADMIN_TOKEN`이 설정된 경우 관리자 화면과 운영 API가 토큰으로 보호됩니다.

## 결론

사용자 페이지와 관리자 페이지는 분리하는 것이 맞습니다. 지금 웹 대시보드는 기능이 빠르게 늘면서 종목 관리, 서버 상태, provider 진단, 백업, 로드맵이 한 화면에 섞여 있습니다. 실제 사용자는 매일 종목과 알림만 보면 되고, 운영 기능은 자주 쓰지 않습니다.

다만 지금 바로 인증 체계를 크게 만들기보다는, 먼저 화면과 라우트를 분리하고 이후 관리자 보호를 붙이는 순서가 안전합니다.

## 분리 전 화면 문제

- 사용자가 자주 보는 정보와 개발/운영 정보가 같은 화면에 있습니다.
- 모바일 화면에서 등록, 감시 종목, 알림 기록 외 영역이 계속 늘어납니다.
- 백업 삭제, 복구, 서버 상태 같은 기능은 일반 사용자 행동과 성격이 다릅니다.
- 앱스토어/플레이스토어 방향으로 가면 사용자 앱에는 운영 기능이 들어가면 안 됩니다.

## 목표 구조

| 영역 | 주소 후보 | 대상 사용자 | 포함 기능 |
|---|---|---|---|
| 사용자 앱 | `/app` 또는 `/` | 일반 사용자 | 종목 등록, 감시 종목, 알림 기록, 배당 캘린더, 포트폴리오 요약 |
| 관리자 | `/admin` | 운영자/개발자 | 서버 상태, 데이터 모델, provider 진단, KIS 현재가 점검, 백업/복구/삭제, 개발 WBS |
| 모바일 API | `/api/mobile/*` | 앱 클라이언트 | 기기 등록, 기기별 종목, 푸시 토큰 |
| 운영 API | `/api/backups`, `/api/health`, `/api/data-model` | 관리자 화면 | 로컬 운영 상태와 진단 |

## 사용자 페이지에 남길 기능

- 종목 등록
- 기준 미리 확인
- 감시 종목 카드
- 포트폴리오 요약
- 배당 캘린더
- 알림 기록
- 텔레그램 테스트 알림은 개발 중에는 남기되, 앱 출시 전 관리자 영역으로 이동

## 관리자 페이지로 옮길 기능

- 서버 상태
- 접속 주소와 QR 코드
- 데이터 모델 버전과 저장소 건수
- 시세 provider 진단
- KIS 현재가 점검
- 배당 provider 진단
- 백업 생성, 복구, 삭제
- 개발 WBS
- 향후 API 키 상태 점검
- 향후 DB 마이그레이션 dry-run 결과

## 단계별 구현

### 1단계: 라우팅 분리

상태: 완료

- `/`와 `/app`은 사용자 앱으로 둡니다.
- `/admin`은 관리자 화면으로 둡니다.
- 기존 `index.html`은 컴포넌트 구조를 유지하되, 사용자/관리자 섹션을 렌더링 모드로 나눕니다.
- 모바일 하단 탭은 사용자 화면에서만 보여줍니다.

### 2단계: 파일 분리

상태: 후속 정리

- `public/app.js`에서 사용자 렌더링과 관리자 렌더링을 분리합니다.
- 후보 파일:
  - `public/app.js`: 공통 초기화
  - `public/userApp.js`: 사용자 화면
  - `public/adminApp.js`: 관리자 화면
  - `public/apiClient.js`: API 호출 공통

### 3단계: 관리자 보호

상태: 완료

로컬 MVP에서는 아래 중 하나를 선택합니다.

| 방식 | 장점 | 단점 |
|---|---|---|
| 관리자 PIN | 구현 간단, 로컬 사용에 충분 | 강한 보안은 아님 |
| `.env` 관리자 토큰 | API 보호가 쉬움 | 사용자가 토큰 관리 필요 |
| 로컬 PC 전용 관리자 | 외부 접근 차단 단순 | 휴대폰 관리자 접근 어려움 |

초기 추천이던 `.env` 기반 `ADMIN_TOKEN`을 적용했습니다. `/admin` 접속 시 토큰을 브라우저 세션에 저장하고, 운영 API 호출에 `x-admin-token`을 붙입니다. `ADMIN_TOKEN`이 비어 있으면 로컬 개발 편의를 위해 관리자 보호는 꺼진 상태로 동작합니다.

### 4단계: 앱 출시 준비

- 사용자 앱에는 관리자 링크를 노출하지 않습니다.
- 관리자 화면은 웹에서만 접근하게 둡니다.
- 모바일 앱은 `/api/mobile/*`만 사용합니다.
- 백업/복구/환경 진단은 앱이 아니라 로컬 관리자 웹에서만 관리합니다.

## 화면 우선순위

사용자 화면 첫 화면:

1. 포트폴리오 요약
2. 위험 종목
3. 감시 종목
4. 배당 캘린더
5. 알림 기록

관리자 화면 첫 화면:

1. 서버 상태
2. KIS 현재가 점검
3. provider 진단
4. 데이터 저장소 상태
5. 백업/복구
6. 개발 WBS
