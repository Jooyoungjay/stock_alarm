# Stock Alarm

Stock Alarm은 사용자가 매수한 종목의 구매일 이후 최고가를 추적하고, 현재가가 최고가 대비 설정한 비율만큼 하락하면 텔레그램으로 반복 매도 알림을 보내는 MVP입니다.

## 현재 MVP 기능

- 감시 종목을 관리하는 웹 대시보드
- 구매일 이후 최고가 자동 계산
- 이후 새 최고가 자동 추적
- 매수가와 구매일 저장
- 종목별 하락률 기준 설정
- 최고가 대비 하락률, 매수가 대비 손절률, 직접 기준가 알림 방식 선택
- 텔레그램 명령어로 종목 등록/조회/중지/삭제/즉시확인
- 반복 알림 간격 설정
- 텔레그램 테스트 알림
- 즉시 가격 확인
- 수동 테스트 가격 입력
- 종목명/코드 자동완성
- 종목별 마지막 확인 상태, 확인 시각, 오류 메시지 표시
- 로컬 JSON 파일 저장
- 외부 패키지 없이 실행 가능한 Node.js 서버

## 로컬 실행

```powershell
Copy-Item .env.example .env
node src/server.js
```

브라우저에서 아래 주소를 엽니다.

```text
http://localhost:3000
```

3000 포트가 이미 사용 중이면 앱이 자동으로 3001, 3002처럼 다음 포트에서 실행됩니다. 서버 시작 로그에 표시되는 주소를 사용하면 됩니다.

## 텔레그램 설정

1. 텔레그램에서 BotFather를 엽니다.
2. `/newbot`으로 봇을 생성합니다.
3. 발급된 봇 토큰을 `.env`의 `TELEGRAM_BOT_TOKEN`에 넣습니다.
4. 만든 봇에게 `/start` 또는 아무 메시지를 보냅니다.
5. 브라우저에서 `https://api.telegram.org/bot<봇토큰>/getUpdates`를 엽니다.
6. 응답의 `chat.id` 값을 `.env`의 `TELEGRAM_CHAT_ID`에 넣습니다.

필수 `.env` 값:

```text
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
```

봇 토큰은 비밀번호처럼 취급해야 합니다. `.env`는 GitHub에 올라가지 않도록 제외되어 있습니다.

## 텔레그램 명령어

서버가 실행 중이면 봇에게 아래 명령어를 보낼 수 있습니다. 보안을 위해 `.env`에 저장된 `TELEGRAM_CHAT_ID`의 채팅만 처리합니다.

```text
/help
/list
/check
/pause 336260
/resume 336260
/delete 336260
```

종목 등록 예시:

```text
/add 336260 두산퓨얼셀 88779 2026-05-11 high 10
/add 336260 두산퓨얼셀 88779 2026-05-11 loss 5
/add 336260 두산퓨얼셀 88779 2026-05-11 target 93000
```

알림 기준 값:

- `high`: 구매일 이후 최고가 대비 하락률
- `loss`: 매수가 대비 손절률
- `target`: 직접 기준가 이하 도달

명령어 확인 주기는 기본 5초입니다. `.env`에서 `TELEGRAM_COMMAND_POLL_SECONDS`로 바꿀 수 있습니다.

## 종목 코드

현재 MVP는 여러 시세 provider를 순서대로 시도합니다.

기본 순서:

```text
naver,stooq,alphavantage,yahoo
```

`.env`에서 순서를 바꿀 수 있습니다.

```text
QUOTE_PROVIDERS=naver,stooq,alphavantage,yahoo
```

provider 역할:

- `naver`: 한국 6자리 종목코드 또는 `.KS/.KQ` 종목 조회
- `stooq`: 미국 종목 조회
- `alphavantage`: Alpha Vantage API 키가 있을 때 사용
- `yahoo`: 마지막 fallback

Alpha Vantage를 사용하려면 `.env`에 API 키를 넣습니다.

```text
ALPHA_VANTAGE_API_KEY=
```

종목 코드 예시:

- 미국 주식: `AAPL`, `TSLA`, `NVDA`
- 한국 주식: `005930`, `000660`, `035720`
- 한국 코스피 접미사도 허용: `005930.KS`, `000660.KS`
- 한국 코스닥 접미사도 허용: `035720.KQ`, `247540.KQ`
- 코드와 이름을 같이 입력해도 6자리 코드가 자동 추출됩니다. 예: `336260 두산퓨얼셀`

운영 서비스로 확장할 때는 약관과 안정성을 확인한 유료 또는 공식 시세 API로 교체하는 것이 좋습니다.

## 종목 등록 도우미

대시보드의 종목 등록 영역에는 종목명/코드 자동완성과 자주 쓰는 종목 예시 버튼이 있습니다.

자동완성 입력 예시:

- `두산`, `퓨얼셀`, `두산 퓨어셀`
- `삼성`, `하이닉스`, `카카오`
- `AAPL`, `Apple`, `엔비디아`

추천 결과를 선택하면 종목 코드와 표시 이름이 자동으로 채워집니다.

- 삼성전자
- SK하이닉스
- 카카오
- Apple
- Tesla

`기준 미리 확인` 버튼을 누르면 등록 전에 현재가와 알림 기준을 확인할 수 있습니다. 조회가 성공하면 종목명, 코드, 현재가, 구매일 이후 최고가, 알림 기준가, 현재 하락률, 사용된 시세 provider가 표시됩니다.

## 구매일 이후 최고가

종목을 등록할 때 `매수가`와 `구매일`을 입력합니다. 앱은 구매일부터 오늘까지의 일봉 데이터를 조회해서 그 기간의 고가 중 가장 높은 값과 매수가 중 더 큰 값을 `구매일 이후 최고가`로 저장합니다.

이후 주기적으로 현재가를 확인하면서 현재가가 기존 최고가보다 높으면 최고가를 새로 갱신합니다. 현재가가 `구매일 이후 최고가` 대비 설정한 하락률만큼 내려오면 텔레그램 알림을 보냅니다.

현재 일봉 provider:

- 한국 종목: Naver 일봉 차트
- 미국 종목: Stooq 일봉 CSV, Yahoo chart fallback

일봉 조회가 실패해도 종목은 등록되며, 화면의 상태와 오류 메시지로 실패 이유를 확인할 수 있습니다. 이 경우 `최고가 재계산`을 눌러 다시 시도할 수 있습니다.

## 알림 테스트

대시보드의 각 종목에는 `테스트 현재가` 입력칸이 있습니다. 실제 시장 가격이 움직일 때까지 기다리지 않고 알림 로직을 검증할 수 있습니다.

예시:

1. 매수가, 구매일, 하락률 기준 5%를 입력해 종목을 등록합니다.
2. 앱이 구매일 이후 최고가를 계산합니다.
3. `테스트 현재가`에 구매일 이후 최고가보다 높은 가격을 입력하면 새 최고가로 갱신됩니다.
4. `테스트 현재가`에 최고가 대비 5% 이상 낮은 가격을 입력하고 `가격 테스트`를 누릅니다.
5. 기준가 이하로 내려간 가격이므로 텔레그램 알림이 전송됩니다.

## 알림 조건 편집

등록된 종목의 `편집` 버튼을 누르면 아래 값을 수정할 수 있습니다.

- 표시 이름
- 하락률 %
- 반복 알림 간격
- 메모

저장하면 기준가와 알림 조건이 즉시 새 값으로 다시 계산됩니다.
